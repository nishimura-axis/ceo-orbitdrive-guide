Telegram Bot + 自動振り返り
セットアップガイド

携帯から音声メモ → AI自動振り返りが届く仕組みを作る

所要時間: 約2〜3時間 対象: Claude Code セットアップ済みの方
1. Bot作成 2. APIキー 3. GCP準備 4. 構築 5. 動作確認 6. デプロイ 7. 振り返り

このガイドを完走すると

  1. 携帯からTelegramに音声/テキストを送る → Obsidian Vaultに自動で書き込まれる
  2. 毎日決まった時間にAIが振り返りを作成 → Telegramに届く
  3. Claude Codeで自分でBotの挙動をカスタマイズできる

何が変わるか

Before（今）

  • Obsidianに書くにはPCを開く必要がある
  • 振り返りは自分で意識してやるしかない
  • 移動中のメモが残せない

After（完了後）

  • 携帯から音声で話すだけで自動保存
  • AIが毎日振り返りを送ってくれる
  • どこにいてもメモできる

全体像

あなたの携帯（Telegram） ↓ 音声やテキストを送信 Telegram Bot（クラウドで常時稼働） ↓ 音声の場合 → 文字起こし（Whisper AI） ↓ テキスト化された内容を整理（Claude AI） ↓ Obsidian Vault に自動書き込み（GitHub経由） ↓ 毎日決まった時間に → AIが今日の活動を振り返り → Telegramにメッセージとして届く
Botはクラウド（Google Cloud Run）で動くので、PCを起動しておく必要はありません。24時間いつでも使えます。

事前準備チェックリスト

  • Obsidian Vault が構築済み（第二の脳構築ガイド完了）
  • Claude Code が使える状態（Claude Codeセットアップガイド完了）
  • GitHub アカウントがある
  • Telegram アカウントを作成済み（スマホ + PC版）
STEP 1 約5分

Telegram Bot を作る

Telegram上で「BotFather」という公式ツールを使って、あなた専用のBotを作ります。

1-1. BotFatherを開く

  1. Telegramアプリ（PCまたはスマホ）を開く
  2. 検索バーに BotFather と入力
  3. 青いチェックマーク付きの公式アカウントをタップ
  4. 「Start」または「開始」ボタンをタップ

1-2. 新しいBotを作成

  1. /newbot と入力して送信
  2. Botの表示名を聞かれるので入力（例: マイアシスタント）
    • 日本語OK、後から変更可能
  3. Botのユーザー名を聞かれるので入力（例: my_assistant_12345_bot）
    • 半角英数字のみ
    • 必ず bot で終わる必要がある
    • 他の人と被らないユニークな名前にする

1-3. トークンを保存

作成が完了すると、BotFatherからこんなメッセージが届きます：

Use this token to access the HTTP API:
7123456789:AAxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
この 7123456789:AA... という文字列がBotトークンです。コピーして安全な場所に保存してください。これはパスワードと同じです。他人に見せないでください。
STEP 2 約15分

APIキーを取得する

Botの中で使うAIサービスの「鍵」を取得します。鍵がないとAIが動きません。

2-1. OpenAI APIキー（音声の文字起こし用）

  1. ブラウザで https://platform.openai.com にアクセス
  2. 「Sign up」でアカウント作成（Googleアカウントでも可）
  3. ログイン後、右上のアイコン →「API keys」をクリック
  4. 「Create new secret key」をクリック
  5. 名前を入力（例: telegram-bot）→「Create secret key」
  6. 表示されたキー（sk- で始まる文字列）をコピーして保存
キーは一度しか表示されません。必ずコピーして保存してください。
料金: 音声文字起こし1分あたり約1円。日常使いなら月数百円程度。
最初に$5のクレジットチャージが必要です。

2-2. Anthropic APIキー（AI振り返り用）

  1. ブラウザで https://console.anthropic.com にアクセス
  2. 「Sign up」でアカウント作成
  3. ログイン後、左メニューの「API Keys」をクリック
  4. 「Create Key」をクリック
  5. 名前を入力（例: telegram-bot）→「Create Key」
  6. 表示されたキー（sk-ant- で始まる文字列）をコピーして保存
料金: AI処理1回あたり数円。日常使いなら月数百円〜千円程度。
最初に$5のクレジットチャージが必要です。

2-3. 取得したキーの確認

ここまでで、以下の3つのキーが手元にあるはずです：

キーどこで取得何に使う
Telegram BotトークンBotFatherBotの認証
OpenAI APIキーplatform.openai.com音声→テキスト変換
Anthropic APIキーconsole.anthropic.comAI振り返り生成
これらのキーは絶対にコードに直接書かないでください。次のステップでClaude Codeが安全に設定します。
STEP 3 約15分

Google Cloudの準備

BotをGoogleのクラウドで24時間動かすための準備です。

3-1. Google Cloudアカウント作成

  1. ブラウザで https://console.cloud.google.com にアクセス
  2. Googleアカウントでログイン
  3. 初めての場合、利用規約に同意
  4. 「無料で開始」をクリック（$300分の無料クレジット付き）
クレジットカード情報が求められますが、無料トライアル中は課金されません。

3-2. 新しいプロジェクトを作成

  1. 画面上部のプロジェクト選択バーをクリック
  2. 「新しいプロジェクト」をクリック
  3. プロジェクト名を入力（例: telegram-bot）
  4. 「作成」をクリック → 作成したプロジェクトを選択

3-3. プロジェクトIDをメモ

ダッシュボードに表示されるプロジェクトID（例: telegram-bot-123456）をメモしてください。後で使います。

## やってほしいこと

### 1. プロジェクト構造の作成
- Python 3.12 + aiogram 3 + FastAPI
- 依存関係管理: requirements.txt
- 環境変数は .env ファイルで管理（.gitignore に含める）
- 上記のキー情報はすべて .env に保存

### 2. Bot の基本機能
- テキストメッセージ受信 → Obsidian Vaultに書き込み
- 音声メッセージ受信 → 以下のフローで処理:
  1. Whisper APIで文字起こし
  2. 文字起こし結果をユーザーに表示し「この内容で保存しますか？」と確認
  3. ユーザーが「はい」→ Vaultに書き込み / 「いいえ」→ 修正を促す
  ※ 音声認識は誤字脱字が多いため、必ず確認ステップを入れる
- Vaultへの書き込み形式:
  - ファイルパス: 00_Inbox/YYYY-MM-DD_HHmm_telegram.md
  - YAMLフロントマター付き（title, type: inbox, created, source: telegram）
  - 書き込み成功したらBotが「保存しました」と返信

### 3. 自動振り返り機能
- 毎日21時に自動実行
- その日にTelegramから送った内容を集約
- Claude API（haiku）で振り返りを生成
- 生成した振り返りをTelegramに送信
- 振り返り内容もVaultに保存（07_DailyLog/YYYY-MM-DD_review.md）

### 4. Cloud Run デプロイ用ファイル
- Dockerfile
- デプロイ用シェルスクリプト
- Cloud Scheduler用の設定手順（READMEに記載）

### 5. セキュリティ
- すべてのキーは環境変数で管理
- .env を .gitignore に含める
- Bot は自分のTelegram IDからのメッセージのみ受け付ける

### 6. README.md
- ローカルでの起動方法
- Cloud Runへのデプロイ手順（初心者向けに丁寧に）

STEP 5 約15分

ローカルで動作確認

クラウドにデプロイする前に、まず自分のPCで動くか確認します。

5-1. Claude Codeに依頼

Botをローカルで起動して動作確認したいです。
起動方法を教えて、一緒に確認してください。

5-2. Telegramでテスト

Botを起動したら、Telegram で以下を試してください：

  1. テキスト送信テスト
    • Botにテキストメッセージを送る
    • 「保存しました」と返ってくればOK
    • Obsidianの 00_Inbox にファイルが作成されているか確認
  2. 音声送信テスト
    • Botに音声メッセージを送る（マイクボタン長押し）
    • 文字起こし結果が表示され「この内容で保存しますか？」と聞かれる
    • 内容が合っていれば「はい」→「保存しました」と返ればOK
    • 誤字があれば「いいえ」→ 正しい内容をテキストで送り直す
うまくいかない場合は、エラーメッセージをClaude Codeに送ってください。
「このエラーが出ました。直してください。」でOKです。

カスタマイズの方法

使い始めてから「こうしたい」が出てきたら、Claude Codeに相談してください。

振り返りの時間を変えたい

自動振り返りの実行時間を21時から22時に変更してください。

振り返りの内容を変えたい

振り返りのプロンプトを変更したいです。
今の振り返り内容に加えて、「明日やるべきこと」も提案してほしいです。

保存先フォルダを変えたい

Telegramから送ったメモの保存先を変更したいです。
「#PJ」で始まるメッセージは30_Projectsに、
それ以外は00_Inboxに保存するようにしてください。

Botの返信を変えたい

保存完了時のBotの返信を変更したいです。
保存したファイル名も返信に含めてください。

トラブルシューティング

Botにメッセージを送っても反応がない

  • Cloud Runが正常に動いているか →「Cloud Runのログを確認して」
  • Webhookが正しく設定されているか →「Webhookの設定を確認して」

音声メッセージが文字起こしされない

  • OpenAI APIキーが有効か確認
  • OpenAIのクレジット残高を確認

振り返りが届かない

  • Cloud Schedulerのログを確認
  • Anthropic APIのクレジット残高を確認
